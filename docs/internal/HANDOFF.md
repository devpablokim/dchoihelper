# deck-factory 완성 핸드오프 (→ 페이블, 새 세션용)

이 문서는 **계획서가 아니다.** 계획은 네(페이블)가 세운다. 이건 맨바닥 새 세션이 프로젝트를 끝까지 끌고 가도록 하는 오리엔테이션 — 현황·자산 지도·제약·미결정 사항이다. 먼저 이 문서를 읽고, 아래 "정본 파일"들을 실제로 열어 파악한 뒤, **계획을 세우고 사용자 승인을 받고 실행**하라.

## 미션

브리프 한 줄 → **발표급** HTML 덱(PNG/PDF/PPTX) 파이프라인의 전체 완성. 현재는 "파이프라인이 끝까지 돌긴 하는 draft" 단계에서 멈춰 있고, 4곳이 껍데기다. 목표는 "발표장 뒷자리에서 안 읽히는 글씨 없음 + 수치엔 출처 강제 + 품질 미달이 조용히 final로 못 나감"을 넘어 **실제 발표급 크래프트를 게이트로 보증**하는 것.

## 실행 전 잠가야 할 결정 (`reference/DECISIONS.md` 6건)

무인 실행이 막히는 지점. 계획 단계에서 **각 항목에 네 추천안을 붙여 사용자에게 한 번에 물어 잠근 뒤** 진행하라. 가장 무거운 blocker는 **A(의미판정 게이트 결정론화)** — H5 타이틀테스트·V5 앵커정합·H4 MECE를 opus 비결정 판정에 둘지 대리지표(예: 앵커명사∩제목명사 코사인<0.30)로 결정론화할지 + arbiter 라벨셋 예산. 이걸 안 정하면 골든셋 라벨링 전체가 막힌다. 나머지 B(수치충돌)·C(매체프로파일 계약변경)·D(이커머스 관할)·E(신설툴링)는 기술 기본값 추천 가능.

## 제약·불변식 (반드시 지켜라)

- **정직성이 이 프로젝트의 생명.** 감사에서 걸린 죄가 전부 "가짜 통과"였다 — 무측정 만점·스텁 export·환각 출처. **스텁으로 초록불 만들기 절대 금지.** 게이트는 실물을 검증해야 하고, 못 하면 정직하게 FAIL/draft.
- **페이블 한도 정지 규칙**: 한도 소진 시 무조건 정지하고 사용자에게 계정 전환 요청. **opus로 폴백 금지.** (메모리 `fable-limit-stop-rule`)
- **워커 분산**: 사용자 선호는 codex 병렬 스폰(동시성 15). WORKERS.md의 produced→gated→merged 규약·파일이 곧 상태 원칙을 따라 세션 끊겨도 재개 가능하게.
- **계획 우선**: 큰 실행 전 계획 세우고 사용자 승인.
- 폰트 자동축소 금지·본문 24pt 하한·수치엔 sourceRef 강제·compose 바이트 결정론 — 기존 품질 불변식 유지.

## 완료 기준 (사용자와 확정하라 — 세 층위)

1. **draft 견고화**: 4개 껍데기 수리(grader var()·export 실렌더·hardfail 강제반려·레이아웃 라우팅), 실제 렌더된 draft 산출.
2. **발표급 final**: 10축 게이트 이식(NEW 패턴 우선 시각앵커) + 골든셋 캘리브레이션으로 90점을 실측 바로 전환 + 사람 마감 루프 E2E.
3. **상용 검증(Phase 5)**: Gamma/Beautiful.ai 블라인드 비교·MBB 체크리스트 통과율.

"전체완성"이 어디까지인지 사용자에게 먼저 확인하고 그 범위로 계획하라.

## 착수 순서 제안 (강제 아님 — 네가 판단)

정직성상 **거짓 초록불부터**(grader var()·export 스텁·hardfail)가 먼저다. 그래야 이후 모든 점수가 근거를 갖는다. 그다음 코퍼스→게이트 이식(시각앵커 축이 최대·최저비용 갭), 마지막 캘리브레이션·상용비교. 상세는 네가 PLAN.md·DECISIONS.md·PATTERN-TAXONOMY.md를 읽고 재구성.
