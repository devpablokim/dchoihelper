# 컨설팅 덱 품질 원칙의 기계 채점 체크리스트화 리서치

## 개요

맥킨지, BCG, 베인(MBB) 등 상위 전략 컨설팅펌의 슬라이드 품질 원칙은 대부분 공식 문서로 유출되어 있지 않으며, 실제로는 (1) 바바라 민토(Barbara Minto)의 피라미드 원칙, (2) MECE 원칙, (3) 액션 타이틀(action title) 관행, (4) think-cell 같은 슬라이드 제작 도구가 강제하는 정렬/서식 규칙, (5) 에드워드 터프티(Edward Tufte)의 데이터-잉크 비율 이론, (6) 발표 디자인 실무 가이드(Guy Kawasaki 10/20/30 규칙, 6x6 규칙, ARL 접근성 체크리스트, assertion-evidence 모델) 등 여러 출처에서 파생된 관행의 집합이다. 이 문서는 이 관행들을 "숫자로 측정 가능한 합격/불합격 기준"으로 변환하고, 각 기준의 측정 방법(파싱 대상, 도구, 임계값)을 명시한다. 목표는 deck-factory 파이프라인에서 생성된 슬라이드(pptx 또는 HTML/이미지 렌더링 결과)를 사람이 보지 않고도 1차로 자동 채점할 수 있는 규칙 세트를 확보하는 것이다.

리서치는 firecrawl 검색/스크레이프로 수행했으며, 유료 논문이나 유출 내부 문서(실제 McKinsey/BCG 내부 스타일 가이드 PDF 원본)는 공개 검색으로 확인되지 않았다 — 이 부분은 "실패"로 명시한다(리스크 섹션 참조). 대신 전직 컨설턴트가 운영하는 교육 블로그(Analyst Academy, Slideworks, Deckary, Slide Science)가 실제 슬라이드 사례를 분해한 2차 자료를 활용했다.

## 핵심 발견

### 1. 피라미드 원칙(Minto)과 액션 타이틀

바바라 민토의 피라미드 원칙은 "결론을 먼저 말하고 근거로 내려간다"는 구조를 요구한다. 이것이 슬라이드 레벨로 내려오면 "액션 타이틀"이 된다 — 슬라이드의 제목 자체가 완결된 문장으로 핵심 결론을 담아야 하며, 독자가 본문을 해석해서 결론을 추론하게 해서는 안 된다. BCG가 1990년대에 이 용어를 만든 것으로 알려져 있다. 실무 규칙은 다음과 같이 수렴한다: 제목은 최대 15단어, 2줄을 넘지 않음, 능동태, 구체적 수치 포함, 접속사 "and"가 들어가면 메시지가 두 개라는 신호. 이른바 "타이틀 테스트"(제목만 훑어도 스토리가 이해되는가)가 검증 방법으로 제시된다.

BCG 실제 슬라이드(2012년 뉴욕시 미디어 산업 보고서) 분해 사례를 보면, 제목-부제-불릿의 3단 피라미드 구조가 시각적으로도 구현된다: 제목이 가장 크고 유일하게 강조색(녹색)이며, 부제는 제목보다 작지만 밑줄로 강조, 본문 불릿은 가장 작다. 즉 피라미드 구조는 폰트 크기 위계와 색상 위계로 시각적으로 강제된다.

### 2. 원 메시지 원칙(One Message Per Slide)

슬라이드 하나에 정확히 하나의 통찰만 담아야 한다는 원칙으로, 액션 타이틀 원칙과 결합된다. 검증 휴리스틱: 제목에 등위 접속사가 있으면 위반 가능성, 본문에 서로 다른 두 개 이상의 결론성 문장(마침표로 끝나는 완결 서술)이 있으면 위반 가능성.

### 3. MECE(Mutually Exclusive, Collectively Exhaustive)

민토가 아리스토텔레스의 논리학에서 원용해 만든 원칙으로, 슬라이드 내 카테고리 분류(막대그래프 항목, 프레임워크 사분면, 목차 섹션 등)가 상호 배타적이고 총체적으로 포괄적이어야 한다. 기계적으로는 완벽한 의미 검증이 어렵지만, 다음과 같은 대리 지표(proxy)로 근사 가능하다: (a) 카테고리 항목들의 수치 합이 명시된 총합과 일치하는가(파이차트, 스택바 차트), (b) "기타(Other)" 카테고리가 전체의 일정 비율(예: 15%)을 넘지 않는가, (c) 동일 슬라이드/차트 내 카테고리 라벨에 중복 어휘가 없는가.

### 4. 데이터-잉크 비율(Tufte)

터프티는 《The Visual Display of Quantitative Information》에서 데이터-잉크 비율을 다음과 같이 정의한다: data-ink ratio = 1 - (삭제해도 정보 손실이 없는 잉크의 비율). 5원칙: (1) 무엇보다 데이터를 보여줘라, (2) 데이터-잉크 비율을 최대화하라, (3) 비데이터 잉크를 지워라, (4) 중복 데이터 잉크를 지워라, (5) 편집을 반복하라. 실무적으로 차트 정크(chart junk)로 분류되는 요소: 3D 효과, 불필요한 그리드라인, 중복 범례(legend와 데이터 라벨 동시 사용), 장식적 그림자/그라디언트. 기계 채점 시에는 "차트 내 순수 장식 요소 개수"를 카운트하는 방식으로 근사한다.

### 5. 폰트 크기 하한

발표(오프라인 라이브) 기준으로 여러 소스가 수렴하는 값:
- 절대 최소 하한: 24pt (ARL 접근성 가이드라인, whitepage.studio 등 다수 소스가 "24pt 미만 금지"로 일치)
- 권장 표준: 28~30pt 이상 (Guy Kawasaki 10/20/30 규칙의 "30pt" 기준과도 일치)
- 제목: 36~44pt, 섹션 헤더: 28~36pt, 본문: 24~30pt, 캡션/각주/출처: 18~20pt(단, 20pt 이상 권장, 절대 하한은 12pt로 낮아지는 것은 온라인 발표/부록 문서에 한함)
- 한 슬라이드 내 사용 폰트 크기 종류는 3종 이하 권장(제목/본문/캡션)
- 온라인 화면 공유 시에는 안전 마진으로 30pt 표준을 권장(창 축소 대응)

### 6. 슬라이드당 단어 수 상한

명시적으로 "몇 단어"라는 절대 표준은 업계 합의가 없지만(브랜드/청중별 편차 인정), 다음 규칙들이 반복적으로 등장하고 서로 정합적이다:
- Guy Kawasaki 10/20/30 규칙: 10슬라이드, 20분, 폰트 30pt 이상 → 30pt 폰트가 물리적으로 한 슬라이드에 들어갈 수 있는 글자 수를 제한하는 방식으로 단어 수를 간접 통제
- 6x6 규칙(또는 4x4/7x7 변형): 불릿 6줄 이하, 줄당 단어 6개 이하 → 슬라이드 본문 총 36단어 이하
- 캡션형 텍스트는 140자 이하 권장(스캔 가능성 기준)
- 액션 타이틀은 15단어/2줄 이하

### 7. 컬러 수 상한

MBB 스타일 정리 자료(Deckary)에 따르면 팔레트는 최대 3~4색으로 제한. 원칙: (a) 강조색은 데이터의 핵심 포인트를 부각하는 데만 사용(장식 금지), (b) 위계는 색상으로 만든다(주요 포인트=브랜드색, 나머지=회색), (c) 배경-텍스트 명암비 확보, (d) 동일 의미(예: 양수=녹색, 음수=빨강)는 덱 전체에서 일관되게 유지. BCG 실사례에서도 기본 팔레트는 녹색 계열 단색조이고, 강조하고 싶은 단 하나의 데이터 시리즈만 파란색으로 예외 처리하는 "1개 예외색" 패턴이 관찰된다.

### 8. 그리드/정렬 규칙

- 모든 슬라이드에서 제목 박스 위치(좌표)가 고정되어 페이지를 넘겨도 움직이지 않아야 함
- 본문은 좌측 정렬(짧은 헤드라인만 중앙 정렬 허용)
- 모든 요소는 공유 그리드 라인에 스냅되어야 하며 정렬 어긋남은 즉시 눈에 띄는 결함으로 간주됨(think-cell, PowerPoint 가이드/그리드 기능이 이를 강제)
- 여백(margin)을 지키고 슬라이드 가장자리까지 요소를 채우지 않음(화이트스페이스 확보)

### 9. 출처 표기와 신뢰성 규칙

정량적 주장에는 반드시 "Source: ..." 형식의 출처 라인이 필요하며, 출처 없는 숫자는 신뢰도를 파괴하는 것으로 취급된다. 모든 슬라이드에 페이지 번호가 있어야 한다.

### 10. 60초 규칙

McKinsey는 컨설턴트에게 슬라이드 하나를 60초 이내에 설명하도록 훈련시킨다고 알려져 있다(2차 출처, 1차 확인 불가). 이는 설계 제약으로 재해석되며, "슬라이드 내용을 60초 내에 소리 내어 읽을 수 있는가"를 발화 속도(분당 130~150단어) 기준으로 환산하면 슬라이드 텍스트 총량이 약 130~150단어를 넘지 않아야 한다는 근사치가 나온다.

### 11. Assertion-Evidence 모델(Michael Alley, Penn State)

컨설팅 업계 밖(공학/과학 발표)에서 나온 모델이지만 액션 타이틀 원칙과 사실상 동일하다: 슬라이드 제목은 완결된 문장(assertion)이어야 하고, 본문은 불릿 리스트가 아니라 시각적 증거(차트/이미지/다이어그램)로 구성해야 한다. 연구에 기반해 청중 이해도와 발표자 신뢰도를 높인다고 주장하며, PowerPoint 템플릿을 무료 배포한다(라이선스는 교육 목적 명시, 상업적 재배포 조건 확인 필요).

### 12. think-cell / MBB 도구가 강제하는 규칙

think-cell은 파워포인트 애드인으로, 차트를 엑셀/Tableau에서 즉시 갱신하고 핵심 포인트를 자동 주석 처리하는 기능을 제공한다. 이 도구의 존재 자체가 업계 표준 관행을 시사한다: (a) 차트 라벨은 범례 대신 데이터 포인트 옆에 직접 배치(범례 왕복 시선 이동 제거), (b) 어젠다/섹션 구분 슬라이드는 자동으로 "현재 위치"를 업데이트, (c) 브랜드 아이덴티티(폰트/색) 준수를 강제하는 템플릿 잠금 기능.

## 재사용 가능 자산(라이선스 명시)

- python-pptx (scanny/python-pptx, GitHub, MIT 라이선스): pptx 파일을 파싱해 텍스트, 폰트 크기, 색상(RGB), 도형 좌표/크기(EMU 단위)를 프로그래밍적으로 추출 가능. 본 리서치가 제안하는 채점기의 핵심 구현 라이브러리로 즉시 재사용 가능. URL: https://github.com/scanny/python-pptx
- hands-on-deck (EveryInc, GitHub, 라이선스 리포에서 별도 확인 필요): pptx를 JSON 패치로 다루는 에이전트 친화적 CLI. 검사/편집 워크플로 설계 참고용. URL: https://github.com/EveryInc/hands-on-deck
- ARL PowerPoint 접근성 체크리스트(Association of Research Libraries, 공개 웹페이지 + PDF): 폰트 24pt 하한, 권장 폰트 목록(Palatino, Georgia, Verdana, Tahoma, Arial, Helvetica), 색상 단독 의미 부여 금지 등 9개 항목의 공개 체크리스트. 기관 공개 자료로 인용 가능. URL: https://www.arl.org/accessibility-guidelines-for-powerpoint-presentations/
- WCAG 2.2 Success Criterion 1.4.3(Contrast Minimum), W3C 공개 표준: 명암비 4.5:1(일반 텍스트), 3:1(18pt+ 또는 14pt+ 볼드 큰 텍스트) 기준을 그대로 슬라이드 텍스트-배경 대비 채점에 활용 가능. URL: https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html
- assertion-evidence.org 무료 PowerPoint 템플릿(Michael Alley, Penn State, 교육용 명시 배포): 액션 타이틀+시각 증거 레이아웃의 참조 템플릿. 상업적 재배포 시 별도 라이선스 확인 필요. URL: https://www.assertion-evidence.org/templates.html
- think-cell 콘텐츠 허브 블로그 글(마케팅 콘텐츠, 저작권은 think-cell 보유, 인용/참고는 가능하나 텍스트 재게시 불가): MECE·피라미드 원칙 적용법 설명. URL은 출처 목록 참조.

각 자산 모두 유료 유출 문서가 아닌 공개 웹 콘텐츠이며, 원문 재배포가 아니라 "규칙 추출"용으로만 사용했다.

## 통합 권고

deck-factory 파이프라인에 다음 3단계 채점기 구조를 권고한다.

1단계: 구조 파서 — python-pptx(또는 HTML 렌더링 파이프라인이면 DOM 파서)로 슬라이드마다 (a) 제목 텍스트, (b) 본문 텍스트 전체, (c) 모든 텍스트 런의 폰트 크기/폰트명/색상, (d) 모든 도형의 bounding box(x, y, w, h), (e) 차트 객체의 시리즈 수·카테고리 라벨·범례 존재 여부를 JSON으로 추출한다.

2단계: 규칙 채점 — 아래 "정량 채점 가능한 합격기준 후보" 20개+를 각각 독립 함수로 구현해 슬라이드별 pass/fail과 근거 수치를 반환한다. 임계값은 본 리서치의 값을 기본값으로 하되 프로젝트별 설정 파일(YAML)로 오버라이드 가능하게 한다.

3단계: 종합 스코어 — 규칙을 카테고리별(구조/타이포/컬러/정렬/데이터시각화/신뢰성)로 그룹핑해 가중합 점수를 산출하고, 특정 치명적 규칙(예: 폰트 24pt 미만, 출처 누락)은 감점이 아니라 즉시 반려(hard fail)로 처리한다.

이미지 렌더링 결과물(PNG/PDF)만 있고 pptx 소스가 없는 경우를 대비해, OCR(Tesseract 등) + 컴퓨터비전 기반 보조 채점기(텍스트 박스 검출로 정렬/여백 측정, 컬러 클러스터링으로 팔레트 수 카운트)를 2차 경로로 준비할 것을 권고한다. 이는 python-pptx 파싱이 불가능한 이미지 전용 산출물에 대한 폴백이다.

## 리스크

- 실제 McKinsey/BCG/Bain의 내부 공식 스타일 가이드 원본 PDF는 인터넷 공개 검색으로 확인되지 않았다(비공개 내부 문서로 추정). 본 리서치의 "MBB 규칙"은 모두 전직 컨설턴트가 운영하는 2차 교육 콘텐츠(Analyst Academy, Slideworks, Deckary, Slide Science)와 think-cell 같은 업계 도구 벤더의 콘텐츠에서 재구성한 것이며, 1차 사료가 아니다. 이 문서의 수치(15단어, 3-4색 등)는 "업계 컨센서스로 수렴하는 값"이지 "공식 규정값"이 아니다.
- "폰트 24pt 하한", "6x6 규칙", "60초 규칙" 같은 숫자 규칙은 출처마다 소폭 다르며(20pt~30pt, 6x6~4x4~7x7 등) 절대적 표준이 아니다. 채점기 임계값은 조정 가능한 설정으로 두어야 하며 고정 상수로 하드코딩하면 안 된다.
- MECE, 피라미드 원칙, "원 메시지" 같은 의미론적 원칙은 자연어 이해가 필요해 완전 자동 채점이 어렵다. 본 문서가 제시하는 측정법은 모두 근사 대리 지표(proxy)이며 오탐/미탐이 발생할 수 있다. 특히 "액션 타이틀인가"는 문장 완결성·능동태·수치 포함 여부로 근사할 수 있지만, "정말 결론을 담고 있는가"는 LLM 기반 2차 판정이 필요할 수 있다.
- think-cell, Guy Kawasaki, Nancy Duarte, assertion-evidence.org 등의 콘텐츠는 마케팅/개인 브랜드 저작물이므로 원문 대량 인용이나 재배포는 저작권 문제가 될 수 있다 — 규칙만 추출해 사용해야 한다.
- 색상 수 상한(3-4색), 폰트 종류 상한(2종) 같은 규칙은 "브랜드 컬러 팔레트 자체가 5색 이상인 조직"에는 그대로 적용하기 어렵다. 채점기에 조직별 화이트리스트 팔레트를 입력받는 구조가 필요하다.

## 정량 채점 가능한 합격기준 후보(20개 이상)

1. 제목 존재 및 길이 상한 — 측정: 슬라이드 title placeholder 텍스트 word count. 기준: 1단어 이상 15단어 이하, 공백 기준 스플릿. 초과 시 fail.
2. 제목 줄바꿈 수 상한 — 측정: 렌더링된 제목 텍스트박스의 줄 수(폰트 크기·박스 너비로 추정 wrap 계산 또는 렌더링 후 픽셀 검사). 기준: 2줄 이하.
3. 제목의 완결 문장 여부(액션 타이틀 휴리스틱) — 측정: 정규식/품사 태깅으로 제목에 동사(서술어)가 포함되는지, 마침표 없이도 주어+서술어 패턴이 성립하는지 판정. 기준: 명사구만으로 끝나는 "토픽 타이틀" 패턴(예: "OO 현황", "OO Analysis")이면 fail.
4. 제목 내 등위접속사 검출(원 메시지 위반 신호) — 측정: 제목 문자열에 "그리고", "and", "및", "와/과" + 두 개의 독립 서술어 패턴이 동시에 있는지 검사. 기준: 검출 시 경고(soft fail), 두 개의 완결 서술어가 모두 있으면 fail.
5. 슬라이드 본문 총 단어 수 상한 — 측정: 슬라이드 내 모든 텍스트 런(제목 제외)의 단어 수 합산. 기준: 40단어 이하 권장, 60단어 초과 시 fail(6x6 규칙 확장 적용).
6. 불릿 줄 수 상한 — 측정: 본문 텍스트 프레임 내 단락(paragraph) 개수. 기준: 6줄 이하.
7. 불릿 1줄당 단어 수 상한 — 측정: 각 단락의 단어 수. 기준: 평균 6~8단어 이하, 개별 최대 12단어 초과 시 fail.
8. 불릿 중첩 깊이 상한 — 측정: 단락의 outline level(python-pptx paragraph.level). 기준: 0~2(3단계) 이하, 3단계(0,1,2,3) 이상이면 fail.
9. 본문 폰트 크기 하한 — 측정: 본문 텍스트 런의 font.size(pt). 기준: 24pt 미만이면 hard fail(오프라인 발표 기준), 온라인 전용 덱은 18pt 하한으로 설정 가능한 변수화.
10. 캡션/출처/각주 폰트 크기 하한 — 측정: 지정된 텍스트 박스(footer, source placeholder)의 font.size. 기준: 12pt 미만 fail, 18pt 미만이면 경고.
11. 슬라이드 내 폰트 크기 종류 수 상한 — 측정: 슬라이드 내 등장하는 고유 font.size 값 개수(set 크기). 기준: 3종 이하(제목/본문/캡션), 초과 시 fail.
12. 슬라이드 내 폰트 패밀리 수 상한 — 측정: 슬라이드 내 고유 font.name 값 개수. 기준: 2종 이하(제목용 1 + 본문용 1), 초과 시 fail.
13. 동일 요소 타입의 폰트 크기 전 슬라이드 일관성 — 측정: 전체 덱에서 title placeholder들의 font.size 표준편차. 기준: 표준편차 0(완전 동일) 권장, 2pt 초과 편차면 fail.
14. 팔레트 색상 수 상한 — 측정: 슬라이드(또는 덱 전체) 내 텍스트/도형/차트에 사용된 고유 RGB 값 개수(그레이스케일 톤 제외, 색상 거리 임계값(ΔE<10)으로 유사색 병합 후 카운트). 기준: 슬라이드당 4색 이하, 덱 전체 팔레트 6색 이하.
15. 강조색 사용 비율 — 측정: 슬라이드 내 강조색(브랜드 액센트 컬러)으로 칠해진 요소 수 / 전체 색상 요소 수. 기준: 10~30% 범위(과도하게 많으면 "강조"의 의미 상실로 fail).
16. 텍스트-배경 명암비(WCAG) — 측정: 텍스트 색상과 배경색의 상대 휘도 기반 대비비(WCAG 공식으로 계산). 기준: 일반 텍스트 4.5:1 이상, 18pt+/14pt+Bold 큰 텍스트 3:1 이상. 미달 시 fail.
17. 데이터-잉크 비율(근사) — 측정: 차트 오브젝트 내 "데이터 표현 요소"(막대, 선, 점, 데이터 라벨) 대 "장식 요소"(3D 효과, 불필요 그리드선, 그림자, 그라디언트 채우기, 중복 범례) 개수 비율. 기준: 장식 요소 개수 0개 권장, 1개 초과 시 감점, 3D 효과 존재 시 hard fail.
18. 차트 범례 대 직접 라벨링 — 측정: 차트에 범례(legend) 객체가 존재하면서 동시에 카테고리가 3개 이하인지 검사. 기준: 카테고리 3개 이하인데 별도 범례를 쓰면 fail(직접 라벨링 권장 위반).
19. 파이차트 카테고리 수 상한 — 측정: 파이/도넛 차트의 데이터 포인트 개수. 기준: 5개 이하 권장, 7개 초과 시 fail(가독성 저하).
20. 그리드 정렬 편차 — 측정: 동일 역할(제목, 본문, 푸터)의 텍스트박스 좌표(x, y)를 전체 슬라이드에서 비교, EMU 단위 표준편차 계산. 기준: 제목 박스 x/y 좌표 편차가 슬라이드 너비의 1% 이내(허용 오차), 초과 시 fail.
21. 요소 간 그리드 스냅 정합성 — 측정: 슬라이드 내 모든 도형의 좌/우/상/하 경계 좌표를 모아 임계값(예: 0.05인치) 이내로 근접한 값들이 그룹을 이루는지 확인(정렬선 클러스터링). 기준: 정렬되지 않은(어느 클러스터에도 속하지 않는) 도형 경계 비율이 10% 미만.
22. 여백(마진) 확보 — 측정: 모든 도형의 bounding box가 슬라이드 가장자리로부터 떨어진 최소 거리. 기준: 슬라이드 짧은 변의 3% 이상 여백 확보, 미만 시 fail.
23. 화이트스페이스 비율 — 측정: 슬라이드 전체 면적 대비 텍스트/도형이 차지하는 면적 비율(bounding box 합산, 겹침 보정). 기준: 콘텐츠 점유율 70% 이하(즉 30% 이상 여백), 초과 시 "과밀" 경고.
24. 출처 표기 존재 여부 — 측정: 슬라이드에 수치/차트/표가 있을 경우 "Source:" 또는 "자료:" 패턴의 텍스트 프레임 존재 여부(정규식). 기준: 수치 포함 슬라이드에서 출처 텍스트 부재 시 hard fail.
25. 페이지 번호 존재 여부 — 측정: 슬라이드 마스터/레이아웃의 page number placeholder 활성화 및 실제 렌더링 여부. 기준: 전체 슬라이드의 100%에 존재해야 함(표지/섹션 구분 슬라이드는 예외 화이트리스트).
26. 발화 시간 근사(60초 규칙) — 측정: 슬라이드 총 단어 수(제목 포함) / 평균 발화 속도(130~150 wpm)로 예상 설명 시간(초) 산출. 기준: 60초(=130~150단어) 이하, 초과 시 경고.
27. 표(table) 밀도 상한 — 측정: 슬라이드 내 표의 (행 수 x 열 수) 셀 개수. 기준: 가시 셀 36개(6x6) 이하 권장, 초과 시 fail 또는 요약 필요 경고.
28. 텍스트 오버플로/자동 축소 발생 여부 — 측정: python-pptx의 text_frame.auto_size 속성이 "SHRINK_TEXT_ON_OVERFLOW"로 설정되어 있고 실제 폰트가 지정값보다 작게 렌더링되는지 확인, 또는 텍스트박스 높이 대비 추정 텍스트 높이 초과 여부. 기준: 오버플로 발생(자동 축소로 폰트가 임계값 24pt 미만이 된 경우) hard fail.
29. 이미지/장식 요소 대 데이터 요소 비율(치장 남용 탐지) — 측정: 슬라이드 내 클립아트/스톡사진/장식용 아이콘(alt text 없음 또는 데이터와 무관) 개수. 기준: 0개(데이터 슬라이드 기준), 1개 이상 존재 시 감점.
30. 색상 의미 일관성(덱 전체) — 측정: 동일 의미로 추정되는 데이터(예: 양수/증가, 음수/감소)에 매핑된 색상이 전체 덱에서 동일한지, 슬라이드 간 RGB 매핑 딕셔너리 비교. 기준: 동일 카테고리에 다른 색이 매핑되면 fail.
31. 대체텍스트(Alt Text) 존재 — 측정: 정보 전달용 이미지/차트 shape의 alt_text 속성 존재 여부(접근성). 기준: 정보성 그래픽의 100%에 alt text 필요, 부재 시 경고.
